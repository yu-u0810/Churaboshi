# Churaboshi Digital Stamp Card

**Next.js + Firebase で構築した、学内夏祭り向けスタンプカードデジタル化システム。**
学生証に記載された NW-7 規格のバーコードをスキャンすることで、紙のスタンプシートを完全にデジタル化。

---

##開発背景・課題

学内夏祭り（屋台３ブース、対象：全校生徒 約400~500名）では、一昨年の初開催から神のスタンプシートを使用していた。

**紙運用の課題：**
- イベント前に全生徒分のシートを印刷・配布する手間
- シートをなくす生徒が一定数いる
- 各ブースでスタッフが手作業でスタンプを押す非効率さ

→**各ブースにスキャナー端末を設置し、学生証バーコードを読み取るだけで管理できるシステムに置換。**
-これにより「学生証を持ち歩く理由」も作れる。

---

## 実装機能

### 1. バーコードスキャン(Quagga2)
- 学生証の NW-7 規格のバーコードをカメラでリアルタイム読み取り
- 複数のバーコードライブラリを実際に比較検証した結果、**レスポンス速度と NW-7 対尾王を両立していた Quagga2 を採用**
- デモ検証で複数端末・複数バーコードの読み取りに成功

**実装で気づいたこと**
レスポンスが良すぎるため、連続スキャンにより、「スキャン成功」画面のすぐあとに「スキャン済みエラー」画面が出てしまい視覚的にうるさくなる問題が発生。運用上問題はないが、利用者ストレス低減のため、スキャン後の一定時間のクールダウンを設ける対策を検討中。

### 2.Firebase によるデータ管理
- データ構造：学籍番号ごとに3ブース分の `true/false` を管理するシンプルな設計
- 約300名が３ブース同時に稼働する想定のため、**リアルタイム性を重視して Firebase を選定**

```
{
  "studants": {
    "学籍番号XXXXX": {
      "booth_1": true,  //わたあめ
      "booth_2": false, //たこせん
      "booth_3": true   //ドリンク
    }
  }
}
```

---

## 技術選定の理由
 
| 技術 | 選定理由 |
|------|---------|
| Next.js | コンポーネント設計とデプロイのしやすさ |
| Firebase | 同時アクセスへのリアルタイム対応、シンプルなDB構造に適している |
| Quagga2 | NW-7 対応 + 複数ライブラリ比較後に最速と判断 |
 
---

## 現状・今後
 
- Firebase バージョン：**2ヶ月後の本番イベントに向けてデモ検証済み**
- フルスタックバージョン（Go + PostgreSQL + Docker）：バックエンドの仕組みを深く理解するために Firebase 版を移植中。  
  Firebaseでは隠れていたデータの流れ（HTTPリクエスト、DB操作）を自分で実装することで、バックエンドの基礎を体感的に学んでいる。
